## TD-017 · DataRow — Mixed `border` and `borderLeft` Shorthand Conflict

| Field | Value |
|---|---|
| **File** | `components/ui/DataRow.tsx` |
| **Location** | Inline style on the root `div` — sets both `border` and `borderLeft` simultaneously |
| **Current behaviour** | Browser applies both properties; `borderLeft` overrides `border`'s left edge. Produces a Chrome DevTools warning about conflicting shorthand and longhand properties on the same element. |
| **Reason deferred** | No visible rendering defect in current screens. Flagged during Sprint 8 visual review. |
| **Impact** | Minor: DevTools noise; potential for unexpected rendering in browsers that process shorthand resolution differently. |
| **Fix** | Replace `border: 1px solid ...` with explicit `borderTop`, `borderRight`, `borderBottom` declarations alongside the existing `borderLeft`, or use `outline` for the hover border. |
| **Proposed sprint** | Sprint 9 polish pass or next time DataRow is touched |
| **Priority** | Low |
| **Blocker** | None |

## TD-018 · Form Inputs Missing `id` / `name` Attributes

| Field | Value |
|---|---|
| **Files** | `components/shared/AddCredentialPanel.tsx`, `components/shared/StartJourneyPanel.tsx`, `components/shared/CreateAmendmentPanel.tsx` |
| **Current behaviour** | `<Input>` and `<Select>` elements rendered without `id` or `name` attributes. Chrome reports missing labels; browser autofill cannot identify fields. |
| **Reason deferred** | No functional impact in the current mock environment. Flagged during Sprint 8 visual review. |
| **Impact** | Accessibility: form inputs are not programmatically associated with their labels (even when labels are visible). Browser autofill is disabled. Screen readers may not announce field purpose correctly. |
| **Fix** | Extend `FormField` to accept an `inputId` prop and pass it to both the `<label htmlFor>` and the child input. Enforce `id` + `name` at the `FormField` level so all future forms get correct accessibility and autofill automatically. |
| **Proposed sprint** | Sprint 9 — implement the `FormField` fix first, then update the three panels |
| **Priority** | Medium (accessibility; required before live user rollout) |
| **Blocker** | None |
